# Transfer_learning
Я протестирую три различных по замыслу алгоритма переноса - DANN, метод моста (Bridging), LSTM с заморозкой слоев - и выясняю, как каждый из них влияет на качество предсказания SoC на реальных данных. 

Предсказание состояния заряда (State of Charge, SoC) литий-ионных батарей необходимо для безопасной эксплуатации электромобилей и точной оценки остаточного пробега. Оптимальным источником для обучения моделей могли бы стать лабораторные данные, поскольку они точны, содержат полные циклы разряда-заряда и эталонный SoC, рассчитанный кулоновским подсчетом. Однако, возникает проблема, модели, обученные на лабораторных данных, плохо работают на полевых. Так, например в статье (https://www.sciencedirect.com/science/article/pii/S2542435123003161#abs0020), которой я вдохновлялся для своего исследования, авторы на реальных данных Audi e-tron(https://doi.org/10.17632/7vdkzpnjgj.2) показали, что главная проблема кроется в сильном влиянии температуры. К примеру показатели BMS зимой и летом нельзя приравнивать из-за изменения химии и кинетики процессов в холоде. 

Но использовать чисто полевые данные для обучения тоже не вариант, так как их трудно собирать в требуемом объеме, сам сбор омрачен неидеальными условиями, что вносят дополнительные погрешности и артефакты, усложняя анализ данных, ко всему прочему, вероятно, модель обученная на одних полевых данных будет работать на других полевых данных, еще хуже чем если бы она обучалась на лабораторных. 

Подобная проблема называется domain gap, а методы ее решения относят к теме трансферного обучения (transfer learning), активно изучаемой в научном сообществе по всему миру. И я в данной работе попытаюсь присоединится к исследованию проблем несоответствия доменов и способов их решения, проведя эксперимент, сравнив между собой три различных по замыслу алгоритма.

Я протестирую три различных по замыслу алгоритма переноса - DANN, метод моста (Bridging), LSTM с заморозкой слоев - и выясняю, как каждый из них влияет на качество предсказания SoC на реальных данных. 

Для анализа в ходе длительного подбора было найдено три наиболее сопоставимых друг с другом датасета (о чем подробно рассказано далее): два лабораторных - Panasonic NCR18650PF и LG HG2, и один полевой - данные с CAN-шины BMWi3.
